# Review Checklist

## Implementation Quality
- [ ] Holes or gaps in the implementation
- [ ] Places to simplify the code
- [ ] Over-engineering or unnecessary complexity
- [ ] Not recreating things that already exist

## Performance
- [ ] Performance issues or degradation
- [ ] Unnecessary allocations or loops

## Code Hygiene
- [ ] TODOs or FIXMEs that shouldn't be committed
- [ ] Debug logs (console.log, print, fmt.Println)
- [ ] Commented-out code
- [ ] Hardcoded test values

## Testing
- [ ] Tests are solid and add value
- [ ] Not over-testing (testing implementation details)
- [ ] Test coverage for new code

## Safety
- [ ] Race conditions
- [ ] Missing error handling
